# Standards Monitoring Task Bank

## Workflow

1. Define recurring or trigger-based monitoring jobs.
2. State the source set, frequency, target artifacts, and output format.
3. Distinguish “no change” logs from “update required” actions.
4. Require evidence and change notes for any revision.
5. Keep a task table with last run, next run, and current status.

## Good Task Types

- standard revision checks
- template applicability review
- recurring FAQ refresh
- user feedback processing
- knowledge-base drift checks

## Quality Checks

- Monitoring tasks are concrete enough to execute.
- Outputs state whether a change occurred.
- Unverified claims are explicitly marked.
