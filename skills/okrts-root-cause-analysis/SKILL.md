# OKRTS Root Cause Analysis

## Models

- OKRTS: Objective -> Key Results -> Tasks -> Steps.
- 5 Whys: move from symptom to direct cause, condition, execution gap, cognition gap, and root cause.
- Problem quadrants: simple/complex crossed with shallow/deep.

## Workflow

1. State the objective and why it matters.
2. Define 2-4 key results that prove progress.
3. Break each key result into tasks and small executable steps.
4. When blocked, run 5 Whys until the root cause is actionable.
5. Classify the problem quadrant.
6. Pick the response: deep decision, automation, documentation, quick handling, or deprioritization.

## Quality Checks

- Steps are concrete enough to start.
- Root cause is not just a restated symptom.
- The chosen response matches problem depth and complexity.
- The plan can be updated after new evidence.
