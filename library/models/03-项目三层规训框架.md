# 项目三层规训框架

## 核心理念

业务类型无非两种：

| 类型 | 特征 | 开发路线 |
|------|------|----------|
| **A类：目标清晰** | 需求明确，可直接撰写 PRD | 文档先行 → 敏捷开发瀑布流 |
| **B类：目标模糊** | 只有 idea，边界不清晰 | 原型先行 → PRD 确定后切换文档先行路线 |

> 核心原则：不做需求模糊的项目。先用最小代价摸清边界，再切换到对应路线。

---

## 三层框架（所有项目均适用）

| 层级 | 名称 | 核心问题 | 关键产出 |
|------|------|----------|----------|
| Layer 1 | 需求对齐层 | 做什么？ | 需求澄清、PRD |
| Layer 2 | 全栈开发层 | 怎么做？ | 原型 / 代码 / 测试 |
| Layer 3 | 测试上线层 | 何时上线？ | 质量验收、部署 |

---

## A类路线：文档先行（需求清晰）

### 流程

```
PRD（术语表 + 交互方式 + 验收标准）
    ↓
技术选型 + Plan + AI Review（两轮以上，人工终审）
    ↓
Sprint 迭代（功能优先）
    ↓
对抗式测试
    ↓
部署上线
```

### PRD 必须包含

```markdown
## PRD

### 术语表
[消除歧义，用详尽语言阐述中文易混淆之处]

### 交互方式
[用户如何与系统交互]

### 验收标准
[边界定义：正常流程 + 异常流程 + 边界特殊情况处理]

---

## PRD.json（任务拆分）
[
  {
    "id": "task-001",
    "title": "xxx",
    "value": false,    // 完成前 false，完成后 true
    "layer": 2,
    "steps": "2min 可完成的原子化步骤"
  }
]
```

### 执行铁律

- AI 将 PRD 拆至 `2min/步` 的原子化颗粒度，此为 AI 稳定完成任务的最小单位
- 技术选型必须由经验最丰富的人把关，不可跳过
- `plan` 与 `review` 由两个 AI 交替进行，两轮以上，人工最终确认
- Git 管理：feature 分支推进、新增 / bugfix 独立分支、小功能点逐一 commit，随时可回滚

---

## B类路线：原型先行（只有 idea）

### 流程

```
模糊 idea
    ↓
快速原型对话（Layer 1 + Layer 2 前端）
    ↓
判断复杂度是否膨胀？
    ├─ 未膨胀 → 继续迭代原型
    └─ 膨胀 → AI 阅读代码生成功能文档 → 切换 A类路线
```

### 触发切换的信号

- 功能点超过 5 个
- 涉及多角色、多系统交互
- 技术选型出现分歧
- 客户/用户反馈原型后需求大幅扩展

### 切换操作

1. AI 阅读现有代码，生成功能文档
2. 基于功能文档撰写正式 PRD
3. 进入 A 类路线

## 三层详解

### Layer 1：需求对齐层

每次需求对接前准备：

```markdown
## 需求对齐总结

### 已知需求
1. ...

### 假设需求
1. ...（基于...推测）

### 待确认点
- [ ] ...
```

> 需求澄清需技术 + 产品面对面或通话，文档无法消除所有歧义。

### Layer 2：全栈开发层

```
MVP 原型（给客户看）→ 客户反馈 → 原型打磨 → 测试数据
        ↓
数据字典 + 数据库设计
        ↓
后端 API + 前端实现（Git 管理，阶段可回滚）
        ↓
单元测试 / 集成测试 / 黑盒测试（开发期并行）
```

### Layer 3：测试上线层

```
对抗式缺陷测试（与开发期测试不同：对抗式暴露缺陷）
        ↓
打包静态资源
        ↓
服务器就绪（客户付费 + 备案确认）
        ↓
部署上线 → 生产环境
```

## Sprint 阶段状态报告模板

```markdown
## 项目状态报告

### 当前里程碑进度
- 里程碑：...
- 进度：X%
- 状态：正常 / 预警 / 阻塞

### 下一阶段计划
- 目标：...
- 关键任务：...
- 预计完成：...

### 潜在风险及应对
| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| ... | ... | ... | ... |

### 资源盘点
- 人力 / 时间 / 预算：...
```

## 项目分类文档映射

| 项目阶段 | 三层框架        | 核心文档              |
| ---- | ----------- | ----------------- |
| 想法   | Layer 1     | 概念文档、市场调研         |
| 论证   | Layer 1     | 可行性分析报告           |
| 启动   | Layer 1     | PRD、项目章程          |
| 规划   | Layer 2     | 技术方案、Sprint Plan  |
| 执行   | Layer 1/2/3 | 代码、会议纪要           |
| 监控   | Layer 3     | 测试及质量保证/进度报告、风险清单 |
| 收尾   | Layer 3     | 验收报告、总结文档         |
