# Document Lifecycle Governance

## Workflow

1. Classify the document type and target audience.
2. Assign required metadata, ownership, and expected lifecycle.
3. Define the state machine: draft, review, approved, archived, deprecated, or local equivalents.
4. Specify review, publication, archival, and reactivation rules.
5. Link documents to projects, standards, and downstream use.

## Outputs

- classification scheme
- lifecycle flow
- status model
- metadata template
- archive and reuse policy

## Quality Checks

- Every formal document has an owner and status.
- Review and archival steps are explicit.
- Reuse does not depend on hidden tribal knowledge.
