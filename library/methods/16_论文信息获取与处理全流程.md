```
信息检索 ──→ 信息筛选 ──→ 关键信息截取 ──→ 逻辑整合
   │             │               │              │
   ▼             ▼               ▼              ▼
 找得到       筛得准       记得精       用得活
```

## 模块一：学术论文信息检索

### 检索渠道分类

| 渠道 | 适用场景 | 特点 |
|------|----------|------|
| **Google Scholar** | 通用检索，标题/作者/关键词 | 覆盖广，被引量可信 |
| **Web of Science** | 高质量期刊检索 SCI/SSCI | 权威但需权限 |
| **CNKI** | 中文文献检索 | 适合中文论文 |
| **Semantic Scholar** | AI 摘要辅助，快速筛选 | 免费，AI 功能强 |
| **Connected Papers** | 找相关论文 | 可视化论文关系图 |

### 检索语法

```google
// 精确标题
"deep learning" AND "medical imaging"

// 限定年份
2020..2024[dp]

// 限定作者
author:"Zhang San"

// 限定期刊
source:"Nature"

# 组合示例
("transformer" OR "attention") AND "medical" AND 2020..2024[dp]
```

### 检索策略

#### 漏斗模型

```
Step 1: 宽泛检索（30-50篇）
   ↓  筛选相关度高的
Step 2: 精选文献（15-20篇）
   ↓  深度阅读摘要
Step 3: 核心文献（5-10篇）
   ↓  精读全文
Step 4: 参考文献（追溯3-5篇）
```

#### 检索词扩展

| 核心词 | 同义词 | 上位词 | 下位词 |
|--------|--------|--------|--------|
| deep learning | neural network, DL | machine learning | CNN, RNN, Transformer |
| sentiment analysis | opinion mining, SA | NLP | aspect-based, emotion detection |

## 模块二：优质论文筛选

### 筛选标准（必读）

| 维度      | 标准                 | 权重  |
| ------- | ------------------ | --- |
| **来源**  | 核心期刊/顶会/知名实验室      | 30% |
| **引用量** | 近3年 >50次，或历史 >200次 | 20% |
| **相关性** | 与研究主题的直接关联度        | 30% |
| **时效性** | 5年内的最新进展           | 20% |

### 快速筛选流程

```
1. 标题筛选：相关 + 有启发 → 保留
2. 摘要筛选：方法明确 + 结果清晰 → 精读
3. 实验筛选：数据可靠 + 方法可复现 → 引用
```

### 预警信号（遇到这些论文快跑）

- ❌ 摘要空洞，无具体方法
- ❌ 引用量极低（<10）且非新领域
- ❌ 实验对比明显不公平（只和很差的比）
- ❌ 作者无相关背景，机构无相关实验室

## 模块三：关键信息截取

### 论文精读模板

```markdown
## 论文信息
- **标题**：
- **作者**：
- **机构**：
- **年份**：
- **期刊/会议**：

## 研究问题
- 核心问题：
- 研究动机：
- 解决的痛点：

## 核心方法
- 创新点：
- 技术路线：
- 关键公式/架构：

## 实验设置
- 数据集：
- 评估指标：
- 对比方法：

## 核心结论
- 主要成果：
- 性能提升：
- 局限性：

## 可借鉴点
- 可以用到我的研究中：
- 方法改进方向：

## 引用建议
⭐⭐⭐⭐⭐（5星：必引用；4星：参考；3星：了解即可）
```

### 信息截取技巧

#### 1. 洗图（图表复用）

```
优先级：原图 > 改绘 > 重绘 > 截图

原则：
- 论文中的图可以引用，但需标注来源
- 推荐自己用 Python/Matplotlib 重绘，数据更可控
- 流程图建议用 Figma/draw.io 重绘
```

#### 2. 洗方法（方法借鉴）

```
拆解步骤：
1. 明确要借鉴的方法的「输入-处理-输出」
2. 提取核心算法逻辑
3. 用自己的语言描述（避免照抄）
4. 结合自己的场景进行适配
```

#### 3. 洗结论（观点引用）

```
引用原则：
- 观点必须忠实原文，不能断章取义
- 数据必须注明来源和年份
- 谨慎引用单一结论，建议多篇对比

引用格式：
据[Author, Year]研究表明，...
```

## 模块四：论文逻辑整合

### 文献综述结构

#### 流派分析法（推荐）

```
## 2.x 研究现状

### 2.x.1 方法派
[流派1] ... [文献1,2,3]
[流派2] ... [文献4,5,6]

### 2.x.2 应用派
[应用领域1] ... [文献7,8]
[应用领域2] ... [文献9,10]

### 2.x.3 总结
现有研究在[方面1]和[方面2]存在不足...
本文针对[不足]进行研究...
```

#### 时间线分析法

```
## 2.x 研究发展脉络

第一阶段（2015-2018）：[基础方法]
[代表文献]

第二阶段（2019-2021）：[快速发展]
[代表文献]

第三阶段（2022-至今）：[最新趋势]
[代表文献]
```

### 逻辑链构建

```
背景（为什么做）
    ↓
现状（别人怎么做）
    ↓
不足（还有哪些问题）
    ↓
动机（本文要解决什么）
    ↓
方法（本文怎么解决）
    ↓
结论（效果怎么样）
```

## 质量检查清单

### 检索阶段
- [ ] 检索词是否有同义词扩展？
- [ ] 检索渠道是否覆盖主流？
- [ ] 时间范围是否合适？

### 筛选阶段
- [ ] 是否有明确的选择标准？
- [ ] 是否区分了「精读」和「略读」？
- [ ] 预警信号是否检查？

### 截取阶段
- [ ] 关键信息是否结构化记录？
- [ ] 图片/方法是否标注来源？
- [ ] 创新点是否清晰提炼？

### 整合阶段
- [ ] 是否有明确的逻辑主线？
- [ ] 是否避免了「文献罗列式」综述？
- [ ] 是否有自己的分析和点评？
