# Solo Scrum Vibe Workflow

## Workflow

1. Define the sprint cadence and delivery window.
2. Select only a small set of backlog items with visible user value.
3. Write acceptance criteria before implementation.
4. Use AI for drafting, coding assistance, documentation, and review, while keeping human judgment on priority and release decisions.
5. Maintain a daily log with done, next, and blockers.
6. Reserve the end of the sprint for integration, review, and retrospective.

## Expected Artifacts

- sprint goal
- selected backlog
- task breakdown
- daily log
- review notes
- retro actions

## Quality Checks

- Each sprint ends with something demonstrable.
- Blockers do not disappear into chat history.
- AI output is reviewed before release.
