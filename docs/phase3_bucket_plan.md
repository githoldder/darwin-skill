# 第三阶段分桶计划

## 目标

对第二阶段之后剩余的 75 个资产做去向判断，避免所有内容都继续向常驻 skill 挤压。

## 分桶结果

- 继续 skill 化：19
- 保留 reference：40
- 应脚本化：13
- 待归档：3

详见 `registry/phase3_asset_buckets.json`。

## 四类含义

### 1. 继续 skill 化

这类资产已经具备稳定触发场景、可执行流程和复用价值，适合进入后续蒸馏批次。

优先顺序建议：

1. 工作流类
2. 研究与写作类
3. 文档治理类

### 2. 保留 reference

这类资产信息量大、背景性强，适合作为长文参考，而不适合直接进入 active skill。

处理方式：

- 保留在 `library/`
- 后续按主题归并
- 被 active skill 按需引用，而不是直接常驻

### 3. 应脚本化

这类资产通常是稳定文本变换、固定格式生成或机械性处理。

处理方式：

- 先抽取输入、输出、参数
- 再考虑合并为一组文本处理脚本或小型 CLI
- 让 skill 负责调度，脚本负责确定性执行

### 4. 待归档

这类资产当前独立价值偏弱、内容较薄或用途过窄，不值得继续占据活跃层。

处理方式：

- 先归档，不急着删除
- 若未来真实任务再次证明价值，再重新激活

## 下一步建议

第三阶段完成后，下一轮建议直接处理：

1. 从 `继续 skill 化` 中再选 6-8 个，做第三批标准化 skill。
2. 从 `应脚本化` 中选一组同类文本处理资产，设计第一个 `text-transform` 脚本族。
3. 把 `待归档` 资产写入正式 archive 记录。
