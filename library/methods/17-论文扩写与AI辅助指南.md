# 论文内容扩写与 AI 辅助指南

> 基于高质量信息输入的内容扩写方法论，解决「AI 味重」「逻辑空洞」「信息堆砌」三大痛点

## 核心问题诊断

| 痛点 | 根因 | 解决思路 |
|------|------|----------|
| AI 格式化味太重 | 提示词缺少「人味」约束 | 引入风格锚定 + 口语化指令 |
| 逻辑空洞，车轱辘话 | 输入信息不足，AI 只能炒冷饭 | 先「补货」再「扩写」 |
| 扩写后质量下降 | 只扩不补，信息密度稀释 | 扩写 = 补充 + 重构 + 深化 |

---

## 扩写工作流（核心方法论）

### Step 1: 补充输入（必做！）

**原则**：扩写前必须确保有**足够的高质量原材料**，否则 AI 只能生产垃圾。

#### 信息补充 SOP

1. **明确扩写目标**
   - 要扩写哪个段落？
   - 缺少什么信息？（数据？案例？对比？）

2. **按场景补充信息**
   - 📊 **调研报告** → 参考 [[18_行研信源获取SOP]]
   - 📚 **学术论文** → 检索高质量文献（见 [[16_论文信息获取与处理全流程]]）
   - 💡 **产品分析** → 补充竞品数据、用户反馈

3. **结构化输入格式**

```markdown
## 扩写任务

### 目标段落
[原文]

### 缺少的信息
1. 缺少 XX 数据支撑
2. 缺少 XX 案例对比

### 已收集的补充材料
- [材料1]：...
- [材料2]：...
```

### Step 2: 生成扩写（提示词模板）

#### 模板 A：学术论文扩写

```
请基于以下材料，扩写以下段落。要求：
1. 保持学术严谨性，避免车轱辘话
2. 每个论点必须有论据支撑
3. 使用[金字塔原理]（请参考004_阅读中的《金字塔原理》书籍）结构：背景 → 问题 → 分析 → 结论
4. 禁止车轱辘话来回说
5. 适当使用连词体现逻辑链（因为...所以...，然而...，此外...）

## 原文
[待扩写段落]

## 补充材料
[Step 1 收集的材料]

## 风格要求
- 正式书面语，但避免「首先」「其次」「最后」堆砌
- 允许使用复杂句式，但必须有信息增量
- 保留原文核心观点，只扩展论证深度
```

#### 模板 B：调研报告扩写

```
请将以下要点扩展为完整段落。要求：
1. 结论先行，每个段落首句必须是核心观点
2. 数据必须有来源标注
3. 使用「总-分-总」结构
4. 避免「由于...因此...」这种无脑因果

## 核心观点
[一句话概括]

## 支撑要点
- 要点1：...
- 要点2：...

## 补充数据/案例
[数据或案例]

## 风格
报告体，客观数据驱动，避免主观形容词堆砌
```

#### 模板 C：商业文案扩写

```
请将以下卖点扩写为有说服力的文案。要求：
1. 场景化：代入用户使用场景
2. 数字化：用具体数字代替「很多」「不错」
3. 差异化：对比竞品/传统方案
4. 人话：像真人写的，不要 AI 格式化

## 核心卖点
[一句话]

## 目标用户
[用户画像]

## 对比参照
[竞品或传统方案的缺点]

## 扩写要求
300-500字，有画面感，有节奏感
```

## 痛点专项解决方案

### 问题 1：AI 味太重

**原因**：提示词缺少风格约束，AI 默认输出「正确的废话」

**解决**：添加风格锚定

```
风格锚定（必加）：
- 禁止使用「首先」「其次」「最后」序列词（除非真正列举步骤）
- 禁止使用「综上所述」「可以看出」「不难发现」等套话
- 允许使用短句增强节奏
- 使用具体案例代替抽象论述
- 适当使用口语化表达（如「说白了」「本质上」）
```

### 问题 2：逻辑空洞

**原因**：输入信息不够，AI 只能炒冷饭

**解决**：分步扩写

```
Step 1: 先让 AI 列出「可以从哪些角度展开」
Step 2: 逐个角度补充材料
Step 3: 再进行扩写
```

### 问题 3：信息稀释

**原因**：只扩不补，原有信息被稀释

**解决**：保持信息密度

```
扩写原则：
- 每扩写 100 字，必须新增至少 1 个信息点
- 不能只是换说法，要加新内容
- 新增内容必须是「补充」而非「重复」
```

## 场景化提示词库

### 场景 1：文献综述扩写

```
请为以下研究主题撰写文献综述。要求：
1. 按「流派/方法」分组，而非按文献罗列
2. 每组先总结共性，再点评差异
3. 指出当前研究空白，引出你的工作

主题：[主题]
已读文献：
- [文献1]：核心观点...
- [文献2]：核心观点...
- [文献3]：核心观点...

结构要求：
- 研究现状
- 流派划分
- 不足与空白
- 本文切入点
```

### 场景 2：方法介绍扩写

```
请扩写以下技术/方法介绍。要求：
1. 背景（为什么需要）- 原理（怎么做的）- 应用（能做什么）
2. 使用流程图/伪代码说明原理
3. 对比类似方法，突出差异化优势

方法名：[方法]
核心原理：[一句话]
对比方法：[类似方法]

输出结构：
1. 背景与动机
2. 核心原理
3. 实现流程
4. 优势与局限
5. 应用场景
```

### 场景 3：数据分析扩写

```
请分析以下数据，撰写分析段落。要求：
1. 描述数据趋势
2. 解释背后原因
3. 推断未来走向
4. 用具体数字而非「增长」「下降」

数据：
[数据1]: 数值 + 变化
[数据2]: 数值 + 变化
[数据3]: 数值 + 变化

输出要求：
- 每个数据点必须有解读
- 数据之间必须有联系
- 结论必须有数据支撑
```

## 质量检查清单

### 扩写完成后的自检

- [ ] **信息增量**：新增内容是否带来新信息？
- [ ] **逻辑连贯**：段落之间是否有明确的逻辑链？
- [ ] **风格自然**：是否像人写的，而非 AI 堆砌？
- [ ] **信息密度**：是否在有效篇幅内最大化信息量？
- [ ] **来源标注**：数据/引用是否有来源？

### AI 味检测

以下词汇出现 >3 次，判定为「AI 味重」：
- 首先、其次、最后、综上所述
- 不难看出、可以看出、显而易见
- 持续稳定、显著提升、不断优化
