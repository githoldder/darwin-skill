# 📘 SOP：基于 AI 与 Figma 的矢量图表高效生产规范

| **版本**   | **1.0**                           | **编制人**  | **Gemini**              |
| -------- | --------------------------------- | -------- | ----------------------- |
| **适用场景** | 需求分析、UI/UX 原型绘制、技术方案架构图、UML 用例图生成 | **核心工具** | LLM (AI), Figma, SVG 代码 |

---

## 1. 流程概述 (Process Overview)

本规范旨在定义从**原始文本需求**出发，利用 AI 进行逻辑建模并生成 SVG 源代码，最终在 **Figma** 中完成二次视觉加工与多格式导出的完整路径。

## 2. 角色与技能要求 (Roles & Skills)

- **需求分析能力：** 能够清晰描述业务参与者与交互逻辑。
    
- **AI 提示词工程：** 熟练使用 PlantUML 语法作为中介逻辑。
    
- **Figma 基础操作：** 掌握解组（Ungroup）、自动布局（Auto Layout）与导出设置。
    

---

## 3. 标准作业程序 (Step-by-Step Procedure)

### 第一阶段：需求建模与逻辑核查 (Modeling)

**操作目标：** 将模糊的需求转化为严密的 UML 逻辑结构。

1. **输入指令：** 使用《UML 逻辑转换模板》对 AI 发起会话。
    
2. **关键检查项：**
    
    - 确认 **Actor（参与者）** 是否包含外部系统或定时任务。
        
    - 确认 **Include（包含）** 与 **Extend（扩展）** 的指向是否符合逻辑。
        
    - 确认 **Note（注释）** 是否涵盖了核心算法或计费规则。
        

### 第二阶段：SVG 源代码生成 (Generation)

**操作目标：** 获得可被 Figma 直接解析的矢量代码。

- **执行标准：**
    
    - 要求 AI 使用 `ellipse`（用例）、`path/line`（关系线）进行编码。
        
    - **强制要求：** 每个用例内部必须包含 `text` 标签填写的**业务详情**。
        
    - **分组要求：** 必须使用 `<g>` 标签按功能模块（Package）封装，便于后期批量移动。
        

### 第三阶段：Figma 渲染与二次创作 (Refinement)

**操作目标：** 完成从“草图代码”到“专业设计稿”的飞跃。

1. **导入：** 直接 `Ctrl + C` SVG 源码，在 Figma 中 `Ctrl + V`。
    
2. **解构：** 选中图形，执行 `Ctrl + Shift + G` 多次，直到图层树中文字与形状分离。
    
3. **排版美化：**
    
    - **对齐：** 使用 `Alt + V / H` 进行等距分布。
        
    - **样式：** 根据业务重要性调整 Fill（填充色）和 Stroke（边框粗细）。
        
    - **响应式：** 建议对功能块执行 `Shift + A` (Auto Layout)，确保修改文字后容器自动适应。
        

### 第四阶段：成果导出 (Exporting)

**操作目标：** 输出符合不同媒介要求的产物。

- **PNG (2x/3x)：** 用于 PPT 汇报、企业微信/钉钉沟通。
    
- **SVG：** 用于在线文档（如 Notion/飞书）或作为前端开发素材。
    
- **PDF：** 用于正式的需求说明书附件。

## 4. 关键快捷键与工具指南 (Toolkit)

|**动作**|**快捷键 (Win)**|**快捷键 (Mac)**|**备注**|
|---|---|---|---|
|**粘贴代码**|`Ctrl + V`|`Cmd + V`|直接在画布空白处粘贴即可|
|**彻底解组**|`Ctrl + Shift + G`|`Cmd + Shift + G`|至少执行 2-3 次以释放嵌套图层|
|**等比例缩放**|`K`|`K`|解决文字随框缩放的问题（必用）|
|**自动布局**|`Shift + A`|`Shift + A`|容器随内容自适应的基础|

---

## 5. 异常处理与注意事项 (Troubleshooting)

- **文字消失/重叠：** 若粘贴后文字位置偏移，检查源码中 `text-anchor="middle"` 是否丢失，手动在 Figma 居中文字即可。
    
- **线条断开：** SVG 粘贴后是静态路径。若需拖动球体带动线条，建议使用 Figma 插件 **"Connector"** 重新连接。
    
- **复杂逻辑缺失：** 若 AI 生成的图过于简单，请在 Prompt 中增加 **“请细化 [具体功能] 的子用例”**。
    

---

## 6. 持续改进 (Maintenance)

定期更新个人的 **SVG Style Snippets**（风格代码片段），记录下你最满意的色值和字体，在下次要求 AI 生成时直接提供色值，实现“视觉风格一键统一”。
